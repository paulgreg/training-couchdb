Apache CouchDb

  • Base NoSQL orienté document ⇒ "schemaless"
  • Similaire à MongoDb
  • Stockage au format JSON
  • Accès par REST (uniquement)
  • Vues par Map/Reduce écrites en JavaScript (ou PHP, Ruby, Python, Erlang)
  • Encourage la réplication (db locale ↔ db distance)
  • Issue de Lotus Notes (documents, réplication, etc)

Architecture

  • Construit sur Erlang dans un but de tourner sur des petites configurations (téléphone, navigateur, etc)
  • Toujours cette idée de réplication...

Exemples d'utilisation

  • Ubuntu :
    • base par utilisateur contenant des données, préférences d'applications (fichiers, Rhythmbox, TomBoy, etc)
    • synchro via Ubuntu One
  • Sauce Labs : "in the cloud cross-browser testing with Selenium"
  • Et des dizaines d'autres références : CouchDB in the wild

Données

  • Objet JSON
{ id: '123456', name: 'Grégory' }
  • Peut contenir des attachements (fichiers)
  • Révisions (conflits)

Accès REST

  • Create → PUT /maBase/id
  • Read → GET /maBase/id
  • Update → POST /maBase/id
  • Delete → DELETE /maBase/id

En action

curl http://127.0.0.1:5984/
{"couchdb":"Welcome","version":"0.10.1"}
curl -X GET http://127.0.0.1:5984/_all_dbs
[]
curl -X PUT http://127.0.0.1:5984/db1
{"ok":true}
curl -X GET http://127.0.0.1:5984/_all_dbs
[db1]
curl -X DELETE http://127.0.0.1:5984/db1
{"ok":true}
